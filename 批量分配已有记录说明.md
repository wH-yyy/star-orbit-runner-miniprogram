# 批量分配已有记录使用说明

## 功能说明

为了处理在实现自动分配功能之前添加的记录，我们提供了批量分配功能。该功能会找出所有未分配工作人员的记录，并自动按照负载均衡算法分配给工作人员。

## 使用方法

### 方法一：使用测试云函数（推荐）

1. **部署测试云函数**

```bash
cd mini_project/formal-project/cloudfunctions/test-batch-assign
npm install
```

在微信开发者工具中：
- 右键点击 `test-batch-assign` 文件夹
- 选择"上传并部署：云端安装依赖"

2. **运行测试云函数**

在微信开发者工具的云函数控制台中：
- 找到 `test-batch-assign` 云函数
- 点击"测试"按钮
- 查看日志输出，确认分配结果

### 方法二：在云函数控制台直接调用

1. 在微信开发者工具中打开"云开发控制台"
2. 进入"云函数"页面
3. 找到 `staff-api` 云函数
4. 点击"调试"或"测试"
5. 输入以下测试数据：

```json
{
  "action": "audit/batchAssign",
  "onlyPending": false,
  "limit": 1000
}
```

6. 点击"开始测试"按钮
7. 查看返回结果

### 方法三：在小程序或 Web 端调用（管理员功能）

在前端代码中调用：

```javascript
import { callFunction, ensureCloudLogin } from '../cloud'

async function batchAssignUnassignedRecords() {
  try {
    await ensureCloudLogin()
    
    const res = await callFunction({
      name: 'staff-api',
      data: {
        action: 'audit/batchAssign',
        onlyPending: false,  // 是否只分配待审核记录
        limit: 1000          // 每次处理的记录数量
      }
    })
    
    console.log('批量分配结果:', res.result)
    
    if (res.result.code === 200) {
      alert(`批量分配完成！
总记录数: ${res.result.data.total}
成功分配: ${res.result.data.assignedCount}
失败记录: ${res.result.data.failedCount}`)
    }
    
    return res.result
  } catch (error) {
    console.error('批量分配失败:', error)
    alert('批量分配失败: ' + error.message)
  }
}
```

## 参数说明

### `onlyPending`（可选，默认 false）

- `false`: 分配所有未分配的记录（包括待审核、已通过、未通过等所有状态）
- `true`: 只分配待审核状态的记录（status = 0）

**建议**：首次使用时设为 `false`，处理所有历史记录

### `limit`（可选，默认 1000）

- 每次批量处理的最大记录数
- 如果数据库中有超过此数量的未分配记录，可以多次调用
- 建议值：500-2000（取决于工作人员数量和服务器性能）

## 注意事项

### 1. 确保有可用的工作人员

在执行批量分配之前，请确保：
- staff 集合中至少有一个 `status='active'` 的工作人员
- 工作人员账号状态正常

查询方法（在数据库控制台中）：
```javascript
db.collection('staff').where({ status: 'active' }).get()
```

### 2. 分批处理大量数据

如果有大量未分配记录（超过 1000 条）：
- 第一次调用：`limit: 1000`
- 查看返回结果中的 `total` 和 `assignedCount`
- 如果 `total === 1000`，说明可能还有更多未分配记录
- 再次调用该函数，继续处理剩余记录
- 重复直到 `total < limit` 或 `total === 0`

### 3. 查看分配结果

分配完成后，可以在数据库中验证：

```javascript
// 查询已分配的记录
db.collection('RunningRecords')
  .where({
    assignedStaffId: _.exists(true)
  })
  .count()

// 查询未分配的记录
db.collection('RunningRecords')
  .where({
    assignedStaffId: _.exists(false)
  })
  .count()
```

### 4. 查看工作人员的任务分布

```javascript
// 统计每个工作人员的任务数量
db.collection('RunningRecords')
  .aggregate()
  .group({
    _id: '$assignedStaffId',
    count: $.sum(1)
  })
  .end()
```

### 5. 错误处理

如果批量分配失败，可能的原因：
- 没有激活状态的工作人员
- 数据库权限问题
- 云函数超时（处理记录过多）

解决方案：
- 检查工作人员状态
- 减小 `limit` 参数值
- 分多次执行

## 执行步骤建议

### 首次执行完整流程

1. **检查工作人员状态**
   ```bash
   # 在数据库控制台执行
   db.collection('staff').where({ status: 'active' }).get()
   ```
   确保有至少一个 active 工作人员

2. **统计未分配记录数量**
   ```bash
   db.collection('RunningRecords')
     .where({ assignedStaffId: _.exists(false) })
     .count()
   ```

3. **执行批量分配**
   - 使用测试云函数或在云函数控制台调用
   - 参数：`{ "action": "audit/batchAssign", "limit": 1000 }`

4. **检查分配结果**
   - 查看返回的 `assignedCount` 和 `failedCount`
   - 如果有失败记录，查看 `failedRecords` 了解原因

5. **验证分配均衡性**
   ```bash
   # 统计每个工作人员的任务数量
   db.collection('RunningRecords')
     .aggregate()
     .group({
       _id: '$assignedStaffId',
       count: $.sum(1)
     })
     .end()
   ```

6. **工作人员登录测试**
   - 使用不同工作人员账号登录 Web 审核系统
   - 验证每个人只能看到分配给自己的记录

## 定期维护

为了确保所有新记录都被自动分配，建议：

1. **监控未分配记录**：定期检查是否有未分配的记录
2. **自动化处理**：可以设置云函数定时触发器，每小时自动执行一次批量分配
3. **日志分析**：查看云函数日志，了解分配情况和潜在问题
